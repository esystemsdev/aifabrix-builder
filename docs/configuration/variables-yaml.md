# variables.yaml

‚Üê [Configuration](README.md)

Generated by `aifabrix create`. Defines your application. The generator transforms nested keys (e.g. `app.key`, `app.displayName`) into the flat structure used by the deployment API and [application-schema.json](../../lib/schema/application-schema.json). **deploymentKey** is not set in variables.yaml; it is computed (SHA256 of manifest) during JSON generation.

**Required:** app.key, app.displayName, app.description, app.type, port (except external), image.name, image.registry, image.registryMode (except external), build.language.

**Optional:** build.localPort, build.dockerfile, build.context, build.envOutputPath, requires.database, requires.databases, requires.redis, requires.storage, healthCheck, authentication, roles, permissions, repository, startupCommand, runtimeVersion, scaling, frontDoorRouting, deployment.controllerUrl, deployment.environment, externalIntegration, configuration (portal UI).

**Example (minimal):**
```yaml
app:
  key: myapp
  displayName: "My Application"
  description: "Does amazing things"
  type: webapp
image:
  name: myapp
  registry: myacr.azurecr.io
  registryMode: acr
port: 3000
build:
  language: typescript
```

See [env.template](env-template.md), [External integration](external-integration.md), [Secrets and config](secrets-and-config.md).
