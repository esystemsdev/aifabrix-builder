# 149.0 — Setup platform mode: single vs full (Traefik front door + multi-env)

## Purpose

Today **`aifabrix setup`** chooses an **installation profile** (`dev` | `pro`) for **admin passwords only** — not for platform topology. Operators who want a **single-environment** laptop install (direct ports, no path prefixes) must manually toggle `traefik` and `useEnvironmentScopedResources`. Operators who want **miso + dev + tst + pro** on one shared Postgres/Docker stack with a **single front-door port** (`3000 + developer-id × 100`, e.g. **3600** for dev **06**) lack a guided path and hit incorrect Traefik labels when **`remote-server`** is unset.

This plan adds a **platform layout** choice at setup time and wires Builder + Controller behavior end-to-end.

## Naming (avoid collisions)

| Concept | Existing name | **New (this plan)** |
| --- | --- | --- |
| Admin password style during setup | `setupInstallationProfile`: `dev` \| `pro` | **Unchanged** |
| CLI login / run target env | `environment`, `aifabrix run --env dev\|tst\|pro` | **Unchanged** |
| Controller DB environment rows | `miso`, `dev`, `tst`, `pro` | Created per mode below |
| **Platform topology** (this feature) | *(none)* | **`setupPlatformMode`**: `single` \| `full` |

**Prompt labels (recommended):**

| Value | User-facing label |
| --- | --- |
| `single` | **Single environment** — direct ports per app (classic dev isolation) |
| `full` | **Full platform** — one front door (Traefik), miso + dev + tst + pro |

**CLI:**

```bash
aifabrix setup --platform single
aifabrix setup --platform full
```

**Do not** overload `setupInstallationProfile` or `environment` for this distinction.

## Target URL matrix (full mode, developer-id `6`, no remote domain)

Full platform front door publish: **`3000 + 6×100 = 3600`** (developer isolation base port + offset; see `lib/utils/dev-config.js`).

| URL | Service | Controller env | Traefik path | Notes |
| --- | --- | --- | --- | --- |
| `http://localhost:3600/miso` | miso-controller | `miso` | `/miso` | No `/dev` prefix on miso-scoped apps |
| `http://localhost:3600/auth` | keycloak | `miso` | `/auth` | IdP for platform login |
| `http://localhost:3600/data` | dataplane | `pro` | `/data` | `aifabrix run dataplane --env pro` |
| `http://localhost:3600/dev/data` | dataplane | `dev` | `/dev/data` | Requires effective env-scoped resources + `--env dev` |
| `http://localhost:3600/tst/data` | dataplane | `tst` | `/tst/data` | Requires effective env-scoped resources + `--env tst` |

**Single mode** (same dev **06**): use **direct** published ports (e.g. miso **3600**, dataplane **3601**, keycloak **3682**) — `traefik: false`, `useEnvironmentScopedResources: false`, **`platform-controller`** = `http://localhost:3600` (no `/miso` suffix).

## Mode → config.yaml mapping

| `setupPlatformMode` | `traefik` | `useEnvironmentScopedResources` | Controller environments (onboarding) | `platform-controller` (typical) |
| --- | --- | --- | --- | --- |
| `single` | `false` | `false` | `miso` only (optional `dev` only if legacy env set — see miso section) | `http://localhost:{3000+dev×100}` |
| `full` | `true` | `true` | `miso`, `dev`, `tst`, `pro` | `http://localhost:{3000+dev×100}/miso` |

Implementation writes these keys during **fresh install** and **re-install** (reinstall re-applies mode from prompt or CLI). **Update-images** / **wipe-data** do not change mode unless user re-runs setup with `--platform`.

Persist:

```yaml
setupPlatformMode: full   # or single
traefik: true
useEnvironmentScopedResources: true
```

Use existing helpers: [`lib/utils/config-scoped-resources-preference.js`](../../lib/utils/config-scoped-resources-preference.js), [`lib/utils/infra-optional-service-flags.js`](../../lib/utils/infra-optional-service-flags.js).

## Problem validation — `applications-dev-06/docker-compose.yaml`

Current generated dataplane labels (Traefik on, **no** `remote-server`, dev **06**):

```yaml
traefik.http.routers.dataplane.rule=Host(`dev06`) && PathPrefix(`/data`)
```

**Bug:** Browsers use **`http://localhost:3600/...`**, not `Host: dev06`. Declarative URL expansion already **falls back to localhost** when `${REMOTE_HOST}` cannot expand ([`buildTraefikPublicBaseIfApplicable`](../../lib/utils/url-declarative-public-base.js) lines 164–165), but **compose Traefik labels still use bare `dev06`**.

**Expected after fix (full mode, localhost, no remote-server):**

```yaml
traefik.http.routers.dataplane.rule=Host(`localhost`) && PathPrefix(`/data`)
# optional: Host(`127.0.0.1`) as second router or HostRegexp — pick one approach in implementation
```

With env-scoped dataplane (`--env dev`, effective scoping on):

```yaml
PathPrefix(`/dev/data`)
```

### 5. Dataplane instances (full mode) — three containers, three registrations

**Yes — full platform requires up to three dataplane Docker containers** on the shared Traefik host, each bound to a **different controller environment** and **different ingress path**:

| Run `--env` | Controller env (`environment` row) | Traefik `PathPrefix` (effective scoping) | Compose router/service key | Example container (dev 06) |
| --- | --- | --- | --- | --- |
| `pro` | `pro` | `/data` | `dataplane` | `aifabrix-dev06-dataplane` |
| `dev` | `dev` | `/dev/data` | `dev-dataplane` | `aifabrix-dev06-dev-dataplane` |
| `tst` | `tst` | `/tst/data` | `tst-dataplane` | `aifabrix-dev06-tst-dataplane` |

Miso stores applications per environment: `@@unique([environmentId, key])` on `Application` — the same app key **`dataplane`** may exist in **pro**, **dev**, and **tst** with separate credentials/deployments ([`schema.prisma`](../../../aifabrix-miso/packages/miso-controller/src/database/prisma/schema.prisma)).

**Per env, Builder must (in order):**

1. **Register** (or rotate) `dataplane` in that **environment key** — `checkApplicationExists(controllerUrl, envKey, 'dataplane')` / `registerApplication` targeting **`envKey`**, not only `config.environment`.
2. **Deploy** manifest to that environment — `deployApp('dataplane', { envKey })` (or equivalent override today missing on deploy path).
3. **Run** locally — `aifabrix run dataplane --env <pro|dev|tst>` so compose gets scoped container name, Traefik labels, `MISO_ENVIRONMENT`, and Redis/kv scoping ([plan 120](Archive/120-environment-scoped_resources_schema.plan.md)).
