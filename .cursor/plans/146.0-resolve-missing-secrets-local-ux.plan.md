## Problem (repro: `hubspot-demo`)

From `/workspace/aifabrix-training`:

```text
aifabrix resolve hubspot-demo
✖ Error in resolve command:
   Missing secrets: kv://hubspot-demo/clientSecret
   Secrets file location: …/secrets.local.yaml …
   Run: aifabrix resolve hubspot-demo to generate missing secrets.

ℹ Run "aifabrix doctor" for environment diagnostics.
```

**Root causes:**

1. **Stale `env.template`** — Wizard left OAuth lines; `repair --auth bearerKey` updated `*-system.json` to `apikey` + `kv://hubspot-demo/apiKey` but **repair merge keeps orphan lines** not in `buildExpectedByKey` (e.g. active `KV_*_CLIENTSECRET=kv://hubspot-demo/clientSecret` while `clientId` is commented).
2. **Misleading fix hint** — Telling the user to run `resolve` again does not help when the missing key is **not required** by the system file (only by a leftover template line).
3. **Wrong resolve environment** — [`lib/cli/setup-utility-resolve.js`](../../lib/cli/setup-utility-resolve.js) always passes `'docker'` to `generateEnvFile`, which applies Docker host/port transforms inappropriate for **local integration** resolve (no container).
4. **Doctor footer** — [`handleCommandError`](../../lib/utils/cli-utils.js) always appends `aifabrix doctor`, which suggests infra/Docker diagnostics even for a **local secrets** failure.

### Expected user-facing fixes (hubspot-demo)

Either:

- Comment out obsolete OAuth lines in `integration/hubspot-demo/env.template`:

  ```bash
  # KV_HUBSPOT_DEMO_CLIENTSECRET=kv://hubspot-demo/clientSecret
  ```

- Or set only the secret the system file needs:

  ```bash
  aifabrix secret set hubspot-demo/apiKey "<private-app-token>"
  ```

After plan **repair-prune-env**, `aifabrix repair hubspot-demo` should remove or comment stale OAuth `KV_*` lines automatically.

## Goals

1. **Actionable errors** — Per missing `kv://` ref: show `env.template` path, the matching `KV_*` line to comment, and `aifabrix secret set <path> "<value>"` (path derived from ref, no hardcoded app names).
2. **Detect stale refs** — When missing paths are not in `authentication.security` / keyvault effective set for the system file, say so (“leftover from OAuth wizard; comment line or run repair”).
3. **Local resolve** — `aifabrix resolve <integration>` uses environment **`local`** (host ports from dev config), not **`docker`**.
4. **Matrix truth** — [cli-output-command-matrix.md](../../.cursor/rules/cli-output-command-matrix.md) gains an **Execution** column: **local** | **docker** | **online** (and combinations where applicable).
5. **No Docker noise** on local-only secret failures — doctor hint gated or replaced with secrets-focused next actions.

## Non-goals

- Auto-generating OAuth secrets on `--force` when auth is `apikey` only.
- Rewriting Plan 139 in-memory resolve for `up-platform` / `up-miso` / `up-dataplane`.

## Design

### A. Resolve environment selection

Add helper (e.g. `lib/utils/resolve-env-mode.js`):

```javascript
/**
 * @param {string} appPath - Resolved directory from getResolveAppPath
 * @returns {'local'|'docker'}
 */
function resolveGenerateEnvEnvironment(appPath) {
  // integration/* → local
  // builder keycloak|miso-controller|dataplane → docker (unchanged)
  // other builder apps → docker (today) or local per application.yaml — document in matrix
}
```

Wire in [`setup-utility-resolve.js`](../../lib/cli/setup-utility-resolve.js):

```javascript
const environment = resolveGenerateEnvEnvironment(appPath);
const envPath = await secrets.generateEnvFile(appName, undefined, environment, options.force, { ... });
```

Success line (human): gray note when `environment === 'local'` — “Resolved for local use (no Docker port rewrite).”

### B. Missing secrets error payload

Extend throw site in [`secrets-env-content.js`](../../lib/core/secrets-env-content.js) `resolveKvReferences` (optional structured suffix, still parseable by formatter):

- `envTemplatePath` (absolute)
- `appName` / `systemKey`
- For each missing ref: `kvPath`, suggested `envVarName` (from template line scan), `inSystemAuth: boolean`

Keep **RFC**: no resolved secret values in the message.

Update [`cli-secrets-error-format.js`](../../lib/utils/cli-secrets-error-format.js):

| Block | Content |
| --- | --- |
| Missing list | Unchanged bullets |
| Template | `env.template:` absolute path |
| Fix A | `Comment out in env.template:` `KV_*=kv://...` (from file) |
| Fix B | `aifabrix secret set hubspot-demo/apiKey "<your-value>"` (dynamic path) |
| Stale OAuth | If `clientId`/`clientSecret` missing but system auth is `apikey` only → yellow warning + `aifabrix repair <systemKey>` |
| Force | Only if catalog has generator: `aifabrix resolve <app> --force` (not default for integration apikey) |
| Remove | Circular “run resolve to fix resolve” as **only** hint |

[`handleCommandError`](../../lib/utils/cli-utils.js): for `formatSecretsError` hits, use **`formatNextActions`** instead of generic doctor line, or doctor only when error also matches Docker-not-running patterns.

### C. Repair: prune obsolete `env.template` KV lines

In [`repair-env-template.js`](../../lib/commands/repair-env-template.js) `mergeEnvTemplateContent` / new `pruneOrphanKvLines`:

- After merge, drop or comment `KV_*=` lines whose `kv://` path is **not** in `buildExpectedByKey` / effective auth set.
- Prefer **comment** (prefix `# `) to align with Plan 102 user intent.
- Append change: `Removed or commented stale env.template KV lines (not in system authentication)`.

Unit test: apikey system + OAuth lines in template → repair comments `CLIENTSECRET` line, keeps `APIKEY`.

### D. CLI matrix — Execution column

Add column **Execution** after **Profile** (or third column before Manifest roots):

| Code | Meaning |
| ---- | ------- |
| **local** | No Docker required; filesystem + `secrets.local.yaml` (+ optional controller HTTP) |
| **docker** | Requires Docker daemon / container lifecycle |
| **online** | Requires login + controller/dataplane HTTP for primary outcome |
| **local+docker** | Local files first, then Docker (e.g. `build`, `run` builder apps) |
| **local+online** | Local manifest + authenticated API (e.g. `upload`, `test-e2e`) |

**Initial row updates (representative):**

| Command | Execution |
| --- | --- |
| `aifabrix resolve` | **local** (integration); **docker** (platform apps `keycloak` / `miso-controller` / `dataplane`) |
| `aifabrix repair` | **local** |
| `aifabrix validate` (integration) | **local** |
| `aifabrix secret set` | **local** |
| `aifabrix test-e2e` | **local+online** |
| `aifabrix run` / `build` | **local+docker** |
| `aifabrix up-infra` | **docker** |

Audit all leaf rows in the matrix file; default **local** for `int` manifest commands unless documented otherwise.
