# 147.0 — Setup modes: image refresh, `platform-controller`, session-aware auth

## Scope

### In scope

- **aifabrix-builder** only: `aifabrix setup` mode menu (3 choices), `setup-modes.js`, `setup-prompts.js`, `setup.js`, new `platform-controller-url` + `setup-platform-auth`, `up-common.applyUpPlatformForceConfig`, `infra-guided` auth/footer, `config.js`, Jest tests, user docs under `docs/`.
- Image pull (`pullInfraImages` + `pullPlatformImages`) on **reinstall**, **wipe-data**, **update-images**, and **fresh install**.
- Persist **`platform-controller`** (absolute Miso URL) in `config.yaml` on every successful setup path.
- Session-aware auth: keep device token when it matches `platform-controller`; otherwise clear tokens + device login.
- Code quality: `npm run lint:fix` → `npm run lint` → `npm run build` → `npm test`.

### Out of scope

- **aifabrix-miso** plan 185 (admin-secrets dev/pro) — do not block 147.
- Replacing `resolveControllerUrl()` for all CLI commands (upload, wizard, …) — follow-up only.
- Azure Marketplace / cloud install.
- Shared `aifabrix-secrets` writes (setup never modifies shared store).
- Dataplane / miso-controller application code changes.

## Problem (concise)

| # | Issue |
| --- | --- |
| 1 | `clean-files` overlaps *update images* — remove from menu |
| 2 | Only `update-images` pulls Docker images today |
| 3 | `up-platform --force` clears tokens + sets `controller` via localhost default even when user is logged in on Traefik/remote URL |
| 4 | No canonical absolute Miso URL in config for setup footers / auth |

## Goals

1. Remove `clean-files` (code, tests, docs).
2. All three modes + fresh install: **`runSetupImageRefresh()`** before `up-infra` (reinstall: after `stopInfraWithVolumes`).
3. **`platform-controller`** in `config.yaml` — same rules as `computeAppBaseUrl('miso-controller')` ([`lib/cli/infra-guided.js`](../../lib/cli/infra-guided.js)).
4. **Auth gate:** `hasStoredDeviceTokenForController(platformUrl)` → skip token clear + skip redundant login; else clear tokens + `handleLogin` to `platform-controller`.
5. After successful login in setup, **`setControllerUrl(platform-controller)`** so `controller` aligns with `platform-controller` (resolves open question #2).

## Decisions (defaults — implement unless user overrides)

| Open item | Decision |
| --- | --- |
| Installation log `setupMode` | Keep stable ids: `reinstall`, `wipe-data`, `update-images`, `fresh` |
| Align `controller` with `platform-controller` | **Yes** — always `setControllerUrl` after resolve; again after login |
| Image pull on reinstall | **After** `stopInfraWithVolumes`, **before** `startInfraFromConfig` |
